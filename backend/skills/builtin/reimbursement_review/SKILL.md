# 报销审核 — 领域知识

## 场景概述

报销审核是对已填写完成的报销单进行正式审核。输入是一份已经填好的表单，目标是执行严格的合规检查并生成正式审核报告。

## 与报销创建的核心区别

| 维度 | 报销创建 | 报销审核 |
|------|---------|---------|
| 输入 | 原始材料（发票、行程单） | 已填写的报销表单 |
| 重点 | 信息提取和表单填写 | 合规校验和标准比对 |
| 输出 | 填好的表单 + 初步审计 | 正式审核报告 |
| 审计严格程度 | 辅助性检查 | 决定性审核 |
| 是否需要推断类型 | 需要 | 不需要（类型已确定） |
| 是否需要填写表单 | 需要 | 不需要（表单已填写） |

## 审核维度（按重要性排序）

### 1. 金额合规性（最高优先级）
- 各项费用是否超出对应职级和城市的标准
- 总金额是否等于各分项之和（用 `sum_values` + `numeric_compare`）
- 预算余额是否充足（用 `get_budget_balance`）
- 所有数值运算必须使用 calculator 工具

### 2. 票据完整性
- 是否有对应的发票或收据
- 发票金额与报销金额是否一致
- 发票日期是否在出差日期范围内
- 调用 `verify_invoice` 验证发票真伪

### 3. 流程合规性
- 是否有事前审批记录
- 审批人权限是否匹配金额级别
- 是否在规定时限内提交

### 4. 政策合规性
- 差旅标准是否符合职级规定
- 交通方式是否符合级别限制
- 餐补是否与接待用餐重复

## 数据查询要求

审核时**必须独立查询最新数据**，不信赖请求中的任何预计算值：
- `get_user_profile` → 确认报销人职级信息
- `get_expense_standards` → 获取最新费用标准
- `get_budget_balance` → 检查预算余额
- `verify_invoice` → 验证发票有效性（有发票号时）

## 审核结论判定规则

- 所有规则 pass → 结论 pass（"审核通过"）
- 存在 warning 但无 fail → 结论 warning（"有条件通过，需关注"）
- 存在任何 fail → 结论 fail（"审核不通过"）

## 审核报告结构

审核完成后通常需要生成审核报告文档（document_type: audit_report），包含：

```
一、报销单基本信息（单据编号、类型、报销人、科室、金额）
二、审核结论（总体结论 + 审核日期）
三、审核明细（逐条规则的审核结果，异常项详细说明标准值 vs 实际值）
四、处理建议（通过/有条件通过/不通过的后续操作建议）
五、附注（特殊情况说明、政策引用）
```

## 质量保障要点

1. **审核严谨性**: 比创建场景更严格，所有可疑项目必须标记
2. **数值精确性**: 所有金额比较必须使用 calculator 工具
3. **结论可追溯**: 审核结论必须有对应的规则结果支撑
4. **报告专业性**: 生成的审核报告可直接用于审批流程
5. **独立性原则**: 必须独立查询最新标准数据
