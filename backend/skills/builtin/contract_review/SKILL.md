# 合同审核 — 领域知识

## 场景概述

合同审核是对已起草完成的合同进行正式审核，识别风险条款、检查合规性并生成审核意见书。这是合同签署前的最后一道关卡，审核结果直接决定合同能否进入签署流程。

## 审核维度详解

### 一、主体资格审查
- 甲方信息是否与 HRP 系统注册信息一致
- 乙方是否具备相应资质
- 法定代表人或授权代表信息是否有效
- 调用 `get_user_profile` 验证合同经办人权限

### 二、合同金额审查
- 金额大小写是否一致（使用 calculator 精确比对）
- 各付款节点金额之和是否等于总金额（使用 `sum_values` + `numeric_compare`）
- 预付款比例是否超过 30%（使用 `calculate_ratio`）
- 合同金额是否在经办人审批权限范围内：
  - 10万以下：科主任
  - 10-50万：分管院长
  - 50-100万：院长
  - 100万以上：院务会

### 三、付款条件审查
- 付款是否与交付/验收挂钩
- 是否存在无条件付款条款（高风险）
- 尾款比例是否足够保障甲方权益（通常不低于 10%）

### 四、期限与交付审查
- 合同期限是否合理（使用 `date_diff` 计算）
- 交付/验收节点是否明确
- 质保期限是否符合行业标准

### 五、违约条款审查
- 违约金比例是否在合理范围（5%-20%）
- 违约条款是否对等（甲乙双方责任对等）
- 是否覆盖常见违约情形

### 六、知识产权与保密审查
- 知识产权归属是否明确
- 保密条款范围和期限是否合理

### 七、争议解决审查
- 争议解决方式是否明确（仲裁或诉讼）
- 管辖法院/仲裁机构是否有利于甲方

### 八、特殊条款审查
- 是否存在不合理的免责条款
- 是否存在霸王条款（对甲方明显不利）

## 风险评级标准

- **高风险 (fail)**: 必须修改后才能签署
  - 金额大小写不一致、预付款超过 30%、缺少违约条款、超出审批权限、无条件付款
- **中风险 (warning)**: 建议修改，可有条件通过
  - 违约条款不对等、保密条款范围过窄、质保期限偏短、争议管辖不利
- **低风险 (pass)**: 符合要求
  - 条款完整且合规、金额和计算准确

## 数据查询要求

审核时**必须独立查询**：
- `get_user_profile` → 确认合同经办人的审批权限
- `get_budget_balance` → 检查合同金额是否在科室预算内
- 所有金额和比例计算使用 calculator 工具

## 审核意见书结构

审核完成后需要生成审核意见书（document_type: review_opinion），包含：

```
一、合同基本信息（名称、编号、甲乙方、金额、期限）
二、审核结论（总体结论 + 审核日期）
三、审核详情（按维度逐条列出审核结果）
四、风险清单（按高/中/低分级列出风险点）
五、修改建议（针对每个非 pass 项给出具体修改建议）
六、审核人意见（综合评价与签署建议）
```

## 质量保障要点

1. **审核全面性**: 必须覆盖主体、金额、付款、期限、违约、知识产权、争议等全部维度
2. **数值精确性**: 所有金额计算和比例判断必须使用 calculator 工具
3. **风险分级准确**: 高/中/低风险的判定标准必须一致且有据可依
4. **建议可执行性**: 每个风险点的修改建议必须具体明确
5. **独立性原则**: 审核必须独立查询最新标准数据
