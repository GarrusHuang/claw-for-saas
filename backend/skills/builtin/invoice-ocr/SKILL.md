# 发票 OCR 后处理能力

## 概述

OCR 识别结果存在固有误差，Agent 需要具备对 OCR 结果进行后处理和校验的能力。
本 Skill 提供常见 OCR 错误的识别模式和修正策略。

## 金额字段 OCR 错误处理

### 常见识别错误

| 错误类型 | OCR 结果 | 实际值 | 识别特征 |
|---------|---------|-------|---------|
| 0/O 混淆 | 1O0.00 | 100.00 | 数字中出现字母 O |
| 1/l 混淆 | l00.00 | 100.00 | 数字以小写 l 开头 |
| 小数点丢失 | 10000 | 100.00 | 金额异常偏大 |
| 小数点偏移 | 1000.0 | 100.00 | 小数位数异常 |
| 逗号误识 | 1,000 → 1.000 | 1000 | 千分位逗号识别为小数点 |
| 空格插入 | 1 00.00 | 100.00 | 数字中间有空格 |

### 金额修正策略

1. **字符清洗**：去除金额中的非数字字符（保留小数点和负号）
   - 替换 O→0, l→1, I→1
   - 去除空格、制表符
   - 去除货币符号（¥、$、元）

2. **小数点校验**：
   - 发票金额通常保留 2 位小数
   - 如果没有小数点且数值异常大，考虑在倒数第 2 位插入小数点
   - 如果小数位超过 2 位，检查是否为千分位逗号误识

3. **大小写金额交叉验证**：
   - 发票通常同时有大写和小写金额
   - 如果 OCR 识别了大写金额（如"壹佰元整"），与小写金额交叉验证
   - 不一致时以大写金额为准（大写不易 OCR 出错）

4. **税额校验**：
   - 税额 = 不含税金额 x 税率
   - 常见税率：1%、3%、6%、9%、13%
   - 使用 calculator 工具验证：`calculate_arithmetic(不含税金额 * 税率)`
   - 如果计算结果与 OCR 税额不符，标记需人工复核

## 日期字段 OCR 处理

### 日期格式标准化

OCR 可能识别出多种日期格式，需统一为 `YYYY-MM-DD`：

| OCR 结果 | 标准化 | 处理方式 |
|---------|--------|---------|
| 2024年3月15日 | 2024-03-15 | 提取数字组合 |
| 2024/03/15 | 2024-03-15 | 替换分隔符 |
| 2024.03.15 | 2024-03-15 | 替换分隔符 |
| 20240315 | 2024-03-15 | 按位拆分插入分隔符 |
| 2024-3-5 | 2024-03-05 | 补零 |

### 日期合理性检查
- 开票日期不应早于业务发生日期
- 开票日期不应晚于报销提交日期
- 日期年份应在合理范围内（当前年份 ±2 年）
- 月份 1-12，日期 1-31，注意各月天数上限

## 发票号码提取

### 发票号码格式规则
- 增值税普通发票：8 位纯数字
- 增值税专用发票：8 位纯数字
- 电子发票：20 位数字
- 发票代码：10 位或 12 位数字

### 常见提取错误
- 将发票代码和发票号码混淆
- 数字间多余空格或分隔符
- 连续数字被断开识别

### 提取策略
1. 优先查找标注为"发票号码"的字段附近的数字串
2. 检查长度是否符合上述规则
3. 去除非数字字符后验证格式
4. 如有校验码字段，一并提取用于验真

## 发票类型识别

### 通过 OCR 文本识别发票类型

| 关键词 | 发票类型 |
|-------|---------|
| "增值税电子普通发票" | 电子普票 |
| "增值税普通发票" | 纸质普票 |
| "增值税专用发票" | 专票 |
| "增值税电子专用发票" | 电子专票 |
| "机打发票" | 通用机打 |
| "定额发票" | 定额发票 |
| "火车票"/"铁路客票" | 火车票 |
| "航空运输电子客票" | 机票行程单 |
| "出租车发票" | 出租车票 |

### 类型识别优先级
1. 标题区域文字（最可靠）
2. 发票代码前缀规则（辅助判断）
3. 版式特征（字段布局）

## 整体校验流程

```
步骤1: 原始 OCR 结果接收
  ↓
步骤2: 字符级清洗（0/O, 1/l, 空格等）
  ↓
步骤3: 字段类型识别（金额/日期/号码/文本）
  ↓
步骤4: 格式标准化（日期统一、金额小数点）
  ↓
步骤5: 交叉校验（大小写金额、税额验算）
  ↓
步骤6: 合理性检查（日期范围、金额范围）
  ↓
步骤7: 标记置信度（高/中/低）
  - 高：通过所有校验
  - 中：部分字段需人工确认
  - 低：关键字段异常，建议人工复核
```

## 注意事项

- OCR 后处理仅做格式修正和合理性检查，不替代发票验真
- 发票验真必须通过 MCP 工具 `verify_invoice` 调用 HRP 系统完成
- 所有金额相关的计算必须使用 calculator 工具
- 修正后的值应标注为"OCR 修正"，保留原始识别值以便追溯
