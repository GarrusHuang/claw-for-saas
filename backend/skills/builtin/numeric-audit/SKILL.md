# 数值审计能力

## 核心原则：禁止心算

所有数值比较和计算操作**必须**使用 calculator 工具完成，严禁 LLM 自行推算。
这是系统铁律，任何违反都可能导致审计结果错误。

### 为什么禁止心算
- LLM 对浮点数运算存在系统性误差（如 0.1 + 0.2 != 0.3）
- 大数值乘除容易产生幻觉结果
- 日期差值计算涉及月份天数差异，人工推算极易出错
- 审计场景要求 100% 计算准确性，不允许"大概正确"

## 数值比较策略

### 金额比较（超标检测）
1. 获取实际金额和标准金额
2. 调用 `compare_numbers(actual, standard)` 获取比较结果
3. 根据返回的 `result`（greater/less/equal）判断是否超标
4. **不要**直接比较数字字符串，必须转为数值后使用工具

### 浮点数容差处理
- 金额比较默认容差 0.01 元（一分钱）
- 比例比较默认容差 0.001（千分之一）
- 超标判定：实际值 > 标准值 + 容差 时才标记超标
- 边界情况：实际值与标准值差距在容差范围内，视为合规

### 常见比较模式

```
模式1：单项费用超标检测
  步骤：compare_numbers(实际住宿费, 标准住宿费)
  判定：result == "greater" → 超标

模式2：总额预算检测
  步骤：calculate_sum([费用1, 费用2, ...]) → 总额
        compare_numbers(总额, 预算余额)
  判定：result == "greater" → 超预算

模式3：比例合规检测
  步骤：calculate_ratio(部分值, 总值) → 比例
        compare_numbers(比例, 标准比例)
  判定：result == "greater" → 比例超标
```

## 日期与天数计算

### 出差天数计算
1. 调用 `calculate_date_diff(start_date, end_date)` 获取天数差
2. 出差天数 = 天数差 + 1（含首尾两天）
3. 住宿天数 = 出差天数 - 1（最后一天不住宿，除非有特殊说明）
4. 餐补天数 = 出差天数（含首尾）

### 日期格式标准化
- 统一使用 `YYYY-MM-DD` 格式
- 遇到 `YYYY/MM/DD` 或 `YYYY年M月D日` 需先转换
- 跨月/跨年日期差必须用工具计算，不可手动推算

### 按天计费的金额计算
```
住宿费总额 = 住宿标准(元/天) × 住宿天数
  → 使用 calculate_arithmetic("标准 * 天数", "multiply")
餐补总额 = 餐补标准(元/天) × 餐补天数
  → 使用 calculate_arithmetic("标准 * 天数", "multiply")
```

## 预算超支检测

### 检测流程
1. 通过 MCP 工具查询科室预算余额：`get_budget_balance(department_id)`
2. 计算本次报销总额：`calculate_sum([各项费用])`
3. 比较总额与余额：`compare_numbers(报销总额, 预算余额)`
4. 生成审计结论：
   - 余额充足（total < balance）→ pass
   - 余额不足（total > balance）→ warning，标注超支金额
   - 预算耗尽（balance <= 0）→ fail

### 超支金额计算
```
超支金额 = 报销总额 - 预算余额
  → calculate_arithmetic("总额 - 余额", "subtract")
超支比例 = 超支金额 / 预算总额
  → calculate_ratio(超支金额, 预算总额)
```

## 多项费用汇总策略

### 分项累加
- 先逐项验证每笔费用的合规性
- 再使用 `calculate_sum` 汇总所有合规费用
- 超标项单独标注，不影响汇总计算

### 发票金额校验
- 发票金额 = 不含税金额 + 税额
- 使用 `calculate_sum([不含税金额, 税额])` 验证
- 差异超过 0.01 元即标记为异常

## 常见错误与防范

| 错误类型 | 说明 | 防范措施 |
|---------|------|---------|
| 心算错误 | 直接输出计算结果 | 所有计算必须调用工具 |
| 类型混淆 | 字符串"100"当数值比较 | 确保传入数值类型参数 |
| 单位遗漏 | 元/万元混用 | 统一为"元"后再计算 |
| 天数偏差 | 忘记+1或多+1 | 明确首尾是否包含 |
| 累加遗漏 | 少加某项费用 | 列出所有费用项后一次性 sum |
