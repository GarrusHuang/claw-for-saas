# 文档生成能力

## 概述

DocGenAgent 负责根据业务数据和模板生成标准化文档。
本 Skill 提供模板填充、条件分段、格式保持和元数据管理的策略指导。

## 占位符替换策略

### 占位符格式规范

系统模板使用以下占位符格式：

| 格式 | 示例 | 用途 |
|------|------|------|
| `{{field_name}}` | `{{applicant_name}}` | 简单字段替换 |
| `{{field_name\|default}}` | `{{department\|未指定}}` | 带默认值的字段 |
| `{{#section}}...{{/section}}` | `{{#items}}...{{/items}}` | 条件区块 |
| `{{@date_format}}` | `{{@YYYY年MM月DD日}}` | 日期格式化 |

### 替换执行顺序
1. **优先级最高**：`known_values` 中的字段（不可覆盖，直接使用）
2. **Agent 填充值**：UniversalFormAgent 输出的 `filled_fields`
3. **默认值回退**：占位符中定义的 `|default` 值
4. **标记未填**：无任何来源时标记为 `[待填写]`

### 替换校验
- 替换后检查文档中是否仍有未替换的 `{{...}}` 标记
- 统计已替换和未替换的字段数量
- 关键字段（金额、日期、当事方名称）未替换时标记为 error
- 非关键字段未替换时标记为 warning

## 条件区块处理

### 条件区块语法
```
{{#has_advance_payment}}
预付款条款：
甲方应在合同签订后 {{advance_days}} 个工作日内
支付预付款 {{advance_amount}} 元（占合同总额 {{advance_ratio}}%）。
{{/has_advance_payment}}
```

### 处理策略
- 条件为真：保留区块内容，替换其中的占位符
- 条件为假：整个区块（含标记行）全部移除
- 嵌套条件：从内向外逐层处理
- 条件判定依据：
  - 布尔字段：直接判断 true/false
  - 数值字段：非零为真
  - 字符串字段：非空为真
  - 列表字段：非空列表为真

### 常见条件区块场景

| 条件字段 | 适用场景 | 说明 |
|---------|---------|------|
| `has_advance_payment` | 合同 | 是否有预付款条款 |
| `has_warranty` | 合同 | 是否有质保条款 |
| `is_overseas_travel` | 报销 | 是否为境外差旅 |
| `has_attachment` | 通用 | 是否有附件列表 |
| `needs_legal_review` | 合同 | 是否需法务审核意见 |

## 格式保持策略

### 原则
模板的格式（缩进、对齐、编号、层级）在填充后必须保持完整。

### 具体规则

1. **缩进保持**：替换后的文本继承占位符所在行的缩进级别
2. **编号连续**：条件区块移除后，重新计算编号序列
   - "第一条"..."第三条" 中移除第二条后，应调整为"第一条"..."第二条"
3. **表格对齐**：
   - 表格内的占位符替换后保持列宽对齐
   - 数值列右对齐，文本列左对齐
4. **段落完整**：替换值为多行文本时，首行跟随占位符位置，后续行保持相同缩进

### 金额格式化
- 小写金额：使用千分位分隔符，保留 2 位小数（如 1,234,567.89）
- 大写金额：转为中文大写（如 壹佰贰拾叁万肆仟伍佰陆拾柒元捌角玖分）
- 同时出现大小写金额时，必须保持一致（使用 calculator 工具验证数值相等）

## 表格数据注入

### 明细表格生成策略
```
输入数据：
  items: [
    {name: "住宿费", amount: 800, standard: 500, note: "超标已审批"},
    {name: "交通费", amount: 1200, standard: 1500, note: ""},
    {name: "餐补", amount: 300, standard: 300, note: ""}
  ]

生成结果：
  | 序号 | 费用项目 | 金额(元) | 标准(元) | 备注         |
  | ---- | ------- | -------- | ------- | ----------- |
  | 1    | 住宿费   | 800.00   | 500.00  | 超标已审批    |
  | 2    | 交通费   | 1,200.00 | 1,500.00|             |
  | 3    | 餐补     | 300.00   | 300.00  |             |
  | 合计 |         | 2,300.00 |         |             |
```

### 表格生成规则
- 序号自动递增
- 金额列使用 calculator 工具计算合计行
- 空值字段显示为空白，不显示 "null" 或 "N/A"
- 合计行加粗或特殊标记

## 元数据自动填充

### 自动生成的元数据字段

| 字段 | 来源 | 格式 |
|------|------|------|
| 生成日期 | 系统当前日期 | YYYY-MM-DD |
| 生成时间 | 系统当前时间 | HH:MM:SS |
| 文档编号 | 业务类型 + 日期 + 序号 | BX-20240315-001 |
| 制表人 | known_values.user_name | 原值 |
| 审核状态 | pipeline 上下文 | 待审核/已通过/已驳回 |
| 版本号 | 默认 V1.0 | V1.0 |

### 编号生成规则
- 报销单：`BX-{YYYYMMDD}-{序号}`
- 合同：`HT-{YYYYMMDD}-{序号}`
- 审计报告：`SJ-{YYYYMMDD}-{序号}`
- 序号为 3 位零填充（001, 002, ...）

## 输出质量检查

### 生成后必检项目
1. **完整性检查**：
   - 所有必填占位符已替换
   - 条件区块已正确处理（移除或保留）
   - 表格数据行数与输入一致

2. **一致性检查**：
   - 同一字段在文档不同位置的值一致
   - 大小写金额一致
   - 日期格式统一

3. **格式检查**：
   - 无残留的占位符标记 `{{...}}`
   - 无残留的条件标记 `{{#...}}` 或 `{{/...}}`
   - 编号序列连续无断裂

4. **业务检查**：
   - 金额合计正确（使用 calculator 工具验证）
   - 日期逻辑合理（开始日期早于结束日期）
   - 必要签章位置已预留

## 注意事项

- 文档生成是最终输出环节，错误传播影响最大，务必严格校验
- 金额相关的格式化和计算必须使用 calculator 工具
- known_values 中的字段值直接使用，不做任何修改
- 生成结果应同时包含文档内容和 metadata（编号、日期、状态等）
